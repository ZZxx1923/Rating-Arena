# مخطط قاعدة بيانات Supabase لمشروع Arena-KSA

بناءً على تحليل المشروع الحالي، سيتم تصميم قاعدة البيانات في Supabase لتلبية المتطلبات التالية:

*   نظام تسجيل الدخول وإدارة المستخدمين (بما في ذلك أدوار المشرفين).
*   إدارة الموظفين والأقسام.
*   نظام تقييمات/تعليقات الموظفين.

سيتم استخدام Supabase Auth لإدارة المستخدمين والمصادقة، بينما سيتم إنشاء الجداول التالية في Supabase Postgres:

## 1. جدول `users`

سيتم استخدام هذا الجدول لتخزين معلومات المستخدمين، بما في ذلك دورهم (مسؤول/مستخدم عادي). سيتم ربط هذا الجدول بنظام Supabase Auth.

| العمود        | النوع                  | القيود                 | الوصف                               |
| :------------- | :---------------------- | :--------------------- | :---------------------------------- |
| `id`           | `UUID`                  | مفتاح أساسي، فريد      | معرف المستخدم (يتم إدارته بواسطة Supabase Auth) |
| `email`        | `TEXT`                  | فريد، غير فارغ         | البريد الإلكتروني للمستخدم (لتسجيل الدخول) |
| `role`         | `TEXT`                  | غير فارغ، افتراضي: 'user' | دور المستخدم ('admin' أو 'user')     |
| `created_at`   | `TIMESTAMP WITH TIME ZONE` | افتراضي: `now()`        | تاريخ ووقت إنشاء المستخدم           |

## 2. جدول `departments`

يخزن هذا الجدول معلومات الأقسام المختلفة في الشركة.

| العمود        | النوع                  | القيود                 | الوصف                       |
| :------------- | :---------------------- | :--------------------- | :-------------------------- |
| `id`           | `UUID`                  | مفتاح أساسي، فريد      | معرف القسم                  |
| `name`         | `TEXT`                  | فريد، غير فارغ         | اسم القسم                   |
| `created_at`   | `TIMESTAMP WITH TIME ZONE` | افتراضي: `now()`        | تاريخ ووقت إنشاء القسم      |

## 3. جدول `employees`

يخزن هذا الجدول معلومات الموظفين، ويرتبط بجدول الأقسام.

| العمود           | النوع                  | القيود                 | الوصف                       |
| :--------------- | :---------------------- | :--------------------- | :-------------------------- |
| `id`             | `UUID`                  | مفتاح أساسي، فريد      | معرف الموظف                 |
| `name`           | `TEXT`                  | غير فارغ               | اسم الموظف                  |
| `position`       | `TEXT`                  | غير فارغ               | منصب الموظف                 |
| `department_id`  | `UUID`                  | مفتاح خارجي (`departments.id`) | معرف القسم الذي ينتمي إليه الموظف |
| `email`          | `TEXT`                  | يمكن أن يكون فارغًا    | البريد الإلكتروني للموظف (اختياري) |
| `created_at`     | `TIMESTAMP WITH TIME ZONE` | افتراضي: `now()`        | تاريخ ووقت إضافة الموظف     |

## 4. جدول `evaluations`

يخزن هذا الجدول تقييمات وتعليقات المستخدمين على الموظفين.

| العمود             | النوع                  | القيود                 | الوصف                               |
| :----------------- | :---------------------- | :--------------------- | :---------------------------------- |
| `id`               | `UUID`                  | مفتاح أساسي، فريد      | معرف التقييم                |
| `employee_id`      | `UUID`                  | مفتاح خارجي (`employees.id`) | معرف الموظف الذي تم تقييمه       |
| `user_id`          | `UUID`                  | مفتاح خارجي (`users.id`) | معرف المستخدم الذي أجرى التقييم |
| `is_anonymous`     | `BOOLEAN`               | غير فارغ، افتراضي: `FALSE` | هل التقييم مجهول؟               |
| `ratings`          | `JSONB`                 | غير فارغ               | تقييمات الأسئلة (مثال: `{"professionalism": 5}`) |
| `comment`          | `TEXT`                  | يمكن أن يكون فارغًا    | تعليق إضافي على التقييم         |
| `status`           | `TEXT`                  | غير فارغ، افتراضي: 'pending' | حالة التقييم ('pending', 'approved', 'rejected') |
| `rejection_reason` | `TEXT`                  | يمكن أن يكون فارغًا    | سبب الرفض إذا تم رفض التقييم    |
| `created_at`       | `TIMESTAMP WITH TIME ZONE` | افتراضي: `now()`        | تاريخ ووقت إنشاء التقييم        |
| `updated_at`       | `TIMESTAMP WITH TIME ZONE` | افتراضي: `now()`        | تاريخ ووقت آخر تحديث للتقييم    |
